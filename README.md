# 文菲的食堂 - 点餐系统

一个基于纯前端技术实现的简单点餐系统，支持菜品管理、分类管理、购物车功能和订单生成。

## 功能特性

### 1. 点餐功能
- 菜品分类浏览
- 菜品详情查看
- 购物车管理（添加、删除、修改数量）
- 底部固定购物车按钮，方便随时查看
- 订单生成与保存

### 2. 后台管理
- 菜品添加、编辑、删除
- 菜品分类自定义管理（添加、编辑、删除分类）
- 支持本地图片上传（Base64编码存储）
- 菜品数据导入导出（JSON格式）

## 技术栈

- **前端**: HTML5, CSS3, JavaScript
- **数据存储**: localStorage (本地存储)
- **部署**: GitHub Pages

## 快速开始

### 本地运行

1. 克隆或下载项目文件
2. 进入项目目录
3. 启动本地服务器（推荐使用Python的内置服务器）:

```bash
python -m http.server 8000
```

4. 在浏览器中访问: `http://localhost:8000`

### GitHub Pages部署

1. 将项目上传到GitHub仓库
2. 进入仓库设置
3. 在"Pages"部分，选择:
   - 分支: `main` 或 `master`
   - 目录: `/` (根目录)
4. 点击"Save"，等待部署完成
5. 访问生成的GitHub Pages URL

### 点餐端使用
1. 在首页浏览菜品，点击"加入购物车"添加商品
2. 点击底部固定的购物车按钮查看购物车内容
3. 调整商品数量或删除不需要的商品
4. 点击"确认下单"生成订单

### 管理端使用
1. 访问 `http://localhost:8000/admin.html` 进入后台管理
2. 点击"添加菜品"按钮新增菜品信息
3. 使用"添加分类"功能自定义菜品分类
4. 可以编辑或删除已有的菜品和分类

### 后台管理

1. 进入后台管理页面（admin.html）
2. 使用「添加菜品」按钮添加新菜品
3. 使用「导出菜品数据」按钮导出所有菜品信息
4. 使用「导入菜品数据」按钮导入菜品信息（支持JSON格式）
5. 可编辑或删除现有菜品

## 数据结构

菜品数据结构：

```json
{
  "id": 1,
  "name": "菜品名称",
  "price": 24.0,
  "sales": 57,
  "image": "图片URL",
  "description": "菜品描述",
  "category": "菜品分类"
}
```

## 注意事项

1. 本系统使用LocalStorage存储数据，刷新页面后数据会保留，但清除浏览器缓存会丢失数据
2. 建议定期导出菜品数据备份
3. 导入数据时会覆盖现有所有菜品数据，请谨慎操作
4. 图片采用Base64编码存储，建议使用较小的图片文件

## 浏览器兼容性
